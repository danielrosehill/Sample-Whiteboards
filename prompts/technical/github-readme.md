## Prompt

> Take this whiteboard photograph and convert it into a clean diagram optimized for GitHub README files. Remove the physical whiteboard, markers, frame, and any background elements -- output only the diagram on a pure white background with clean edges and no border. Correct any perspective distortion so the output appears as a perfectly straight-on, top-down view. Preserve all the original content, text, and diagrams. When reading handwritten text, infer the correct spelling of technical terms, product names, and proper nouns rather than transcribing handwriting literally (e.g. "Proxmox" not "Proxknox", "Kubernetes" not "Kubernites"). Use GitHub's familiar color palette -- blues (#0969DA), greens (#1A7F37), purples (#8250DF), and neutral greys. Shapes should be clean rounded rectangles with subtle borders. Use a system sans-serif font similar to what GitHub renders. Add simple, recognizable developer icons (git branches, terminal prompts, API endpoints, databases). The result should look like a native GitHub diagram that blends seamlessly into a README -- clean, developer-friendly, and immediately understood by anyone browsing a repo.
